WEB: https://github.com/semihumanbeing/youtube-share

API: https://github.com/semihumanbeing/Youtube-Share-API

사용 기술

Frontend: React.js, Typescript

Backend: Java 17, SpringBoot 3.2, Websocket, Redis, JPA, QueryDsl, MariaDB

Server: Google Cloud Platform, Nginx

  • 비용 절감을 위해 클라우드 서버를 비교하여 결정
  • Websocket, Redis, JWT, Spring security 등 업무하며 사용해 본 백엔드 기술 사용하여 빠르게 개발
  • CI/CD 를 위해 git pull 한 뒤 프로필 적용하여 배포하는 쉘 스크립트를 작성해 크론탭에 적용
  • 회사에서는 사용하지 않던 SSE, QueryDSL을 공부하고 적용
  • 프론트엔드 개발은 처음이라 리액트 강의와 책을 따라하고 자바스크립트 도서 전체를 필사하여 적용

문제해결

  • 채팅, 유투브 비디오 재생, 플레이리스트 동기화 모두에 웹소켓을 사용하다보니 속도 저하로 오작동하는 이슈를 해결하기 위해 HTTP polling, SSE, Long polling 등의 통신 방법을 조사하고 비교하였고 플레이리스트 표시부분을 SSE 를 통해 HTTP 통신으로 돌려 최적화.
  • 플레이리스트의 잦은 조회로 인한 DB 부담을 덜기 위해 Redis를 사용하여 빠르게 내용을 불러올 수 있도록 하였고, Redis의 Queue 를 사용하여 순서대로 곡을 추출하여 재생할 수 있는 구조 설계
